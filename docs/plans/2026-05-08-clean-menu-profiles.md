**Goal:** Keep the GRUB menu clean while making boot behavior explicit, repeatable, and scalable through PartBoot profiles.

**Architecture:** Continue using stock GRUB/shim binaries and own the generated `grub.cfg`. Do not fork GRUB. Move per-ISO decisions into profile metadata, then generate only the entries the profile says should be visible.

**Tech Stack:** Rust 2021, standard library only, generated GRUB2 config, JSON-like profile files generated by PartBoot.

### Task 1: Record GRUB Strategy

**Files:**
- Create: `docs/architecture/grub-strategy.md`

**Step 1: Document decision**

State that PartBoot does not edit or build GRUB from source for now.

**Step 2: Explain why**

Use stock GRUB/shim because it reduces Secure Boot, firmware, filesystem, and maintenance risk.

**Step 3: Define when to revisit**

Revisit source builds only for custom embedded config, custom modules, or a real signing pipeline.

### Task 4: Add Profile Metadata

**Files:**
- Create: `src/profile.rs`
- Modify: `src/main.rs`
- Modify: `src/grub.rs`

**Step 1: Generate profile**

When scanning/extracting an Ubuntu ISO, create:

```text
partboot/profiles/ubuntu-22.04.5-desktop-amd64.profile
```

**Step 2: Store fields**

Store:

```text
name
family
preferred_mode=extracted
fallback_mode=iso_toram
visible_fallback=true
```

**Step 3: Use profile in menu**

Generate preferred entry first and fallback only if `visible_fallback=true`.
Menu labels must stay minimal:

```text
<iso-name>.iso
<iso-name>.iso [Fallback]
```
